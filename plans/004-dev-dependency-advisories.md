## Why this matters

`npm audit` in `/web` reports 2 vulnerabilities (1 critical, 1 moderate). The critical
one is in `vitest` (<3.2.6, GHSA-5xrq-8626-4rwp: arbitrary file read/execute when the
Vitest UI server is listening). Both are **dev-only** — `npm audit --omit=dev` is
clean, so production is unaffected — but a developer running `vitest --ui` on an
untrusted network is exposed, and open advisories rot into CI noise. The repo has a
history of keeping the audit clean (commit `930e3e2` "clear npm audit advisories").
